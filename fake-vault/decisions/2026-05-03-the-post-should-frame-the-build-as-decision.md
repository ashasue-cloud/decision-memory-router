# Decision: The Post Should Frame The Build As Decision

## Decision

The post should frame the build as decision memory, not note search.

## Why

decisions are the context that compounds across roadmaps, strategy, reviews, interviews, and content.

## Options Considered

- Chosen: The post should frame the build as decision memory, not note search.
- Rejected: describing the build as an Obsidian search tool.

## Tradeoff Accepted

narrower technical scope, stronger product point of view.

## What This Affects

- Content briefs.
- Portfolio proof.
- Interview stories about product judgment and AI-assisted work.

## Assumptions

- Decision memory is a sharper product claim than note search.
- The hiring audience will value the decision layer more than broad vault search.
- Narrower scope makes the build easier to judge.

## Missing Data

- Whether readers immediately understand the distinction between decision memory and note search.
- Whether future recall output actually changes a task, not just retrieves a note.
